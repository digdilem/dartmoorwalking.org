# Contributing

Contributions to Dartmoor Walking are welcome and appreciated.

## Before You Start

If you plan a significant change or a new walk, please open an issue first to discuss it, or drop me a line.

## Reporting Bugs

Please include:

- A clear description
- Steps to reproduce
- Logs or screenshots if relevant
- Your environment (OS, version)

## Reporting typos, errors etc

- Please raise an issue describing the problem
- The URL(s) involved
- If you'd like to create a pull request fixing the problem, that would be great - but I'm also happy to action that changes myself.

## Submitting a new walk

I would love to hear about new walks on Dartmoor and, if they are suitable, include them on this site.

You don't need to code your walk's site, but if you can, great - details on that below.

For any submitted information, there are some key rules;

1. You must be the creator or copyright holder of all submitted information - photographs, route and words.
2. The walk must be within Dartmoor National Park
3. If it already exists on another website, please mark that clearly.
4. Please use your real name.

### Submitting a new walk (Pre-coded)

Please use one of the newer walks as a template. The format is mostly:
- One directory named with the date of the walk.
- Within that directory, - "index.md" contains the page, formatting and frontmatter - written in Markdown as per Hugo's syntax, tailored to this site's theme.
- All images would be in JPG format, named 1.jpg, 2.jpg etc and referred to such in the index.md

Ideally, you would clone this repo, build the new page and test the site locally before submitting a pull request with the submission. Please don't change any files outside of `content/post/YYY-MM-DD` with your submission as this may mean the PR is rejected.

Any questions, please drop me a line or raise an issue.

### Submitting a new walk (Infomrmation only)

I'm also happy to convert submissions as well. Just write the walk out and include the photographs. That could be in Word or HTML format, or just as a text file with annotations. Then send them to me! You'll get credit as the author.

## Pull Request Guidelines

- Keep PRs small and focused
- Write clear commit messages
- Update documentation if necessary

## Code Style

Follow the existing style used in the project. This is especially important on the frontmatter
